破損的存取控制

存取控制執行政策，使用戶無法在其預期權限之外行動。失效會導致未經授權的資訊洩露、修改或銷毀所有資料。

lock_open 核心概念

存取控制（通常稱為授權）是網頁應用程式如何授予某些用戶存取內容和功能的權限，同時拒絕其他人。這些檢查在身份驗證之後執行。

破損的存取控制漏洞存在於當用戶實際上可以存取某些資源或執行他們不應該能夠存取的某些操作時。由於其普遍性，它在 OWASP Top 10 2021 清單中躍升至第一位。

常見攻擊向量

攻擊者可以利用這些缺陷存取未經授權的資料或功能。以下是典型的表現形式：

  • link_off
    不安全直接物件參考 (IDOR) 操控 URL ID (e.g., /user/123 to /user/124) 在未驗證的情況下查看其他用戶的個人資料。
  • admin_panel_settings
    特權提升 以一般使用者身份登入時，透過操控隱藏欄位或 API 端點來取得管理員角色。
  • travel_explore
    強制瀏覽 直接透過猜測 URL (e.g., /admin) 存取受限頁面，跳過登入或選單導航。

"如果只需更改 URL 中的 ID 就能查看他人的報稅表，那就是存取控制失效。這雖然簡單，卻危害極大。"

預防 & 緩解

block

默認拒絕

存取控制機制應集中實作一次，並在整個應用程式中重複使用；預設情況下應拒絕所有存取。

dns

模型存取控制

系統必須強制檢查每筆資料的擁有權，不得允許使用者對非其所屬的記錄進行建立、讀取、更新或刪除。

verified_user

伺服器端驗證

不要依賴客戶端驗證。攻擊者可以修改請求（URL、標頭、Cookie）來繞過用戶界面檢查。
