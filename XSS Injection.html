XSS 注入

跨站腳本攻擊（XSS）是網路應用程式中常見的安全漏洞。XSS 允許攻擊者將惡意的客戶端腳本注入到其他用戶查看的網頁中，有效地繞過同源策略。

javascript 核心概念

當應用程式在新網頁中包含未經信任的數據而沒有進行適當的驗證或轉義，或使用可以創建 HTML 或 JavaScript 的瀏覽器 API 以用戶提供的數據更新現有網頁時，就會出現 XSS 漏洞。

機制：攻擊者將惡意腳本（通常是 JavaScript）插入應用程式。當受害者訪問頁面時，腳本在其瀏覽器中執行，使攻擊者能夠竊取 Cookie、會話令牌或其他敏感資訊。

常見攻擊手法

攻擊者利用 XSS 來劫持用戶會話、篡改網站內容或將用戶重定向到惡意網站。以下是三種主要類型：

  • save
    儲存型 XSS（持久性） 惡意腳本永久儲存在目標伺服器上（例如在論壇帖子或評論中）。當受害者查看儲存的內容時，會檢索到惡意腳本。
  • keyboard_return
    反射型 XSS（非持久性） 腳本從網路伺服器反射回來，例如在錯誤訊息或搜索結果中。它通過特製的連結傳遞給受害者。
  • integration_instructions
    DOM 型 XSS 漏洞存在於客戶端代碼而非伺服器端。攻擊載荷是通過修改受害者瀏覽器中的 DOM「環境」來執行的。

「雖然 SQL 注入針對資料庫，但 XSS 針對用戶。它有效地將一個受信任的網站變成了針對其自身訪客的武器。」

預防與緩解

translate

上下文感知編碼

在渲染之前將特殊字元轉換為其 HTML 實體等效項。編碼類型（HTML、JavaScript、URL）必須與數據放置位置相符。

security

內容安全政策（CSP）

實施 CSP HTTP 標頭以限制瀏覽器允許載入資源（如腳本）的來源，減輕 XSS 的影響。

cleaning_services

清理過濾

如果您的應用程式必須接受某些 HTML（例如富文本編輯器），請使用可信任的函式庫（如 DOMPurify）在渲染之前解析和清理 HTML。
