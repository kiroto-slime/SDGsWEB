SQL 注入

SQL 注入（SQLi）是一種網路安全漏洞，允許攻擊者干預應用程式對資料庫進行的查詢。它使攻擊者能夠查看他們通常無法檢索的數據。

database 核心概念

網路應用程式經常使用 SQL（結構化查詢語言）與資料庫進行通訊。如果應用程式接受用戶輸入（如搜索詞或登入 ID）並在沒有驗證的情況下直接將其串聯到資料庫查詢字串中，它就會變得容易受到攻擊。

機制：攻擊者在輸入欄位中插入特殊的 SQL 字元（如 ' 或 "）。這會欺騙資料庫解釋器將用戶輸入視為代碼而非數據，從而改變查詢的邏輯以執行惡意命令。

常見攻擊手法

攻擊者利用 SQL 注入來繞過身份驗證、訪問、修改或刪除資料庫中的數據。以下是典型的表現形式：

  • login
    身份驗證繞過 通過輸入 admin' OR '1'='1 無需密碼即可登入，這使得查詢條件始終為真。
  • table_view
    基於 UNION 的 SQL 注入 使用 UNION 運算符將原始查詢的結果與其他表的結果組合，暴露敏感數據。
  • visibility_off
    盲注 SQL 注入 通過向資料庫詢問真/假問題並觀察應用程式行為或響應時間來推斷資料庫內容。

「數據是現代網路最有價值的資產。SQL 注入就像是因為有人客氣地詢問就把銀行金庫的門打開一樣。」

預防與緩解

code

參數化查詢

也稱為預準備語句。這迫使資料庫區分代碼和數據，確保用戶輸入永遠不會被解釋為命令。

rule

輸入驗證

使用允許的字元、類型和格式的「白名單」。如果欄位期望的是整數（如 ID），請確保只處理整數。

admin_panel_settings

最小權限原則

確保網路應用程式使用的資料庫用戶帳戶僅具有最低必要權限。它不應該能夠刪除表格。
